Usage

In this tab, you can observe and influence the genetic evolution of a population of individuals over successive generations.

Header

Tracks

The Tracks button allows you to choose a track from your personal list.

Toolbar

Generation Counter

It displays the generation number of the population that is currently navigating the circuit. It increases by one each time the entire population has lost, or when time is up.

Time Controller

It displays the playback speed. The double arrows on either side allow you to increase or decrease this speed.
The Play/Pause button allows you to start or pause the race.
The Start Over button clears all data related to the current process. Evolution parameters are reset, and those of the Before the start section become modifiable again.

Selector

The selector allows you to choose between displaying an evolutionary parameters tab and a statistics tab.

Parameters

Before the start:

The parameters in this section are, as its name suggests, only modifiable before the start of evolution. To access it again, it will be necessary to click on the Start Over button.

  • The count slider defines the number of individuals in the population.
  • The sliders for minimum and maximum speed allow you to modify the speed limits of the cars.
  • The Insert button allows you to add one (or more, by repeating the action) of your individuals to the first population of the evolutionary process.

Other Settings:

The parameters in this section are accessible at any time during evolution.

  • The Mutation rate. For more information, see the part of the guide explaining the Genetic Algorithm.
  • The Social Mixing. For more information, see the part of the guide explaining the Genetic Algorithm.
  • The Time slider corresponds to the maximum duration allocated to each generation: when time is up, even if not all cars have been eliminated, the current generation ends.
  • The Include Champion option does not model a natural mechanism, but it is still particularly convenient. When activated, the best individual of each generation is directly inserted into the next one. Thus, the best score never decreases. If the parameters are poorly adjusted and this option is disabled, you could see decreases in the score (for example with a mutation rate of 100%, in which case the scores would be purely random).

Statistics

Note that the data displayed in this section correspond to that of the previous generation: it would be impossible to obtain definitive statistics on the generation currently being evaluated.

The best score is displayed first. It corresponds to the number of tiles traveled by the champion of the previous generation.

Then, a graph shows the curve of best score per generation. You can thus observe the effectiveness of the evolution process. Note that the graph only starts to fill in from generation 2, because two points are necessary to draw the first straight segment. The Show Data button opens a window presenting the graph data in the form of a table.

Below, the average score in the population is presented.

A second graph works like a connected histogram. It gives an overview of the score distribution in the population, expressing for each score the percentage of the population that obtained it.
Again, a show data button is available.

Finally, a view presents the neural network of the current champion. For more information, see the part of the guide on Neural Networks. A Save button allows you to save the champion, and to name it.

When I press a Show Data button, the evolution is paused. Is this normal?

Yes, it pauses automatically. By making sure you don't miss a single moment of the process, we aim to make your experience better. You can restart it using the play button.
